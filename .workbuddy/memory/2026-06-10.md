## 第二轮 BUG 修复与打包 (13:21)

### 根因：localStorage key 冲突 — BUG #3 #4
App.vue 和 NotePanel.vue 使用**同一 key** `sn-notebooks`（TodoPanel 同理 `sn-todolists`），但存储不同数据结构：
- App.vue saveTabData() 只存 `{id, name, color, locked}`（tab 元数据，无 notes）
- NotePanel.vue 存完整数据 `{id, name, notes: [...]}` 
操作链：修改 tab 颜色/锁定/重命名 → saveTabData() 写入无 notes 数据 → 派发 notebooks-updated → NotePanel loadData() 读取 → 所有笔记丢失！

**修复**：saveTabData() 改为**合并模式**——先读已有完整数据，只更新 name/color/locked 字段，保留 notes/todos。

## BUG 修复：新建 Tab 有时点击无效 (16:14)

### 根因
`.sub-tabs` 容器设置了 `overflow-x: auto`（隐藏滚动条）。当 tab 数量较多时，容器产生水平溢出，新添加的 tab 虽然渲染出来了，但被裁剪在滚动区域的不可见部分，用户看不到也点不到。tab 少时（未溢出）则正常，导致了「有时能点、有时不能点」的无规律现象。

### 修复
`addSubTab()` 中，`refreshSubTabs()` 后增加 `nextTick(() => { subTabsEl.scrollLeft = subTabsEl.scrollWidth })`，将容器滚动到最右端，确保新 tab 始终可见可点击。

## BUG 修复：新建 Tab 点击无反应（第二版）(16:21)

### 根因分析
第一版修复（scrollLeft）解决了"看不到"的问题，但用户反馈 tab 可见、hover 变灰正常，**点击却无反应**。

根因在 `switchNotebook`/`switchTodolist` 事件处理器中：
```javascript
if (notebooks.value.find(b => b.id === id)) {
  switchBook(id)
}
```
当 `notebooks.value`（面板本地状态）与 localStorage 数据不一致时，`find()` 返回 undefined，整个处理器**静默失败**——没有日志、没有 UI 反馈，用户感觉就是"按了没反应"。

数据不一致的来源：`App.vue` 的 `saveTabData()` 和 `NotePanel` 的 `saveData()` 操作同一个 localStorage key (`sn-notebooks`)。虽然 JS 单线程保证了不会同时写，但在 Vue 批量更新期间，`notebookList`（tab 元数据）和 `notebooks`（完整数据含 notes）可能出现短暂不同步。

### 修复
1. **NotePanel.vue** & **TodoPanel.vue**：切换处理器增加容错 — 先检查本地状态，找不到则调用 `loadData()` 从 localStorage 重新加载，确保数据一致后再切换
2. **App.vue**：`.tab-bar` 加 `-webkit-app-region: no-drag` 防止 Tauri 拖拽区域干扰点击

## 第八轮 五大修复 (17:00 -> 次日凌晨)

### 1. 导出 Markdown 按钮无反应
**根因**：Tauri WebView 中 `<a download>` 方式下载 blob URL 可能被安全策略拦截，导致点击无任何反馈。
**修复**：`exportMarkdown()` 改为 `async` 函数，优先使用 `navigator.clipboard.writeText()` 将 Markdown 内容复制到剪贴板，并显示 toast 提示「已复制 Markdown 到剪贴板」；同时保留 `<a download>` 作为备选方案。新增 `toast` ref + `showToast()` + `.panel-toast` UI 组件。

### 2. 笔记编辑器默认富文本，移除纯文本模式
**修改**：
- `richMode` 默认从 `false` 改为 `true`
- 移除纯文本 `textarea`（原来的 `v-else` 分支）
- `toggleRichMode()` / `toggleMarkdownMode()` 改为强制激活模式（不再支持 toggle off）
- `loadData()` 中默认启用富文本（除非 `sn-markdown-mode === '1'`）
- 模板中 Markdown 编辑器从 `v-else-if` 改为 `v-else`

### 3. 深色模式显示异常
**根因**：编辑器区域 `background: transparent` 导致 contenteditable 可能继承浏览器默认背景色；部分子元素可能带有浏览器默认的硬编码颜色。
**修复**：
- `.note-editor` 添加 `background: var(--bg-app)`
- `.note-content-editable` 背景从 `transparent` 改为 `var(--bg-app)`
- `.note-textarea` 背景从 `transparent` 改为 `var(--bg-app)`
- `.todo-panel` 添加 `background: var(--bg-app)`
- `.note-content-editable *:not([style*="color"])` 添加 `color: inherit` 确保无内联颜色的子元素继承正确暗色文字

### 4. 待办页筛选浮动按钮移到左侧
**修改**：TodoPanel.vue `.progress-section` 中，将 `.filter-float-btn` 按钮移到 `.progress-bar` 前面；移除 `.filter-float-btn` 的 `margin-left: auto`。

### 5. Tab 栏新标签无法点击
**根因**：`activeSubTabId` 是 `computed` 属性，从 `localStorage` 读取，但 Vue 的 reactivity 系统不会监听 localStorage 变化。点击 tab 时虽然更新了 localStorage，但 `activeSubTabId` 不会重新计算，导致 active 样式不更新。
**修复**：
- `activeSubTabId` 从 `computed` 改为 `ref(0)`
- 新增 `syncActiveSubTabId()` 函数从 localStorage 同步到 ref
- `switchSubTab()` 中直接设置 `activeSubTabId.value = id`
- `watch(activeTab, syncActiveSubTabId)` 和 `watch(notebookList/todolistList, syncActiveSubTabId)` 保持同步
- `notebooks-updated` / `todolists-updated` 事件监听器中调用 `syncActiveSubTabId()`
